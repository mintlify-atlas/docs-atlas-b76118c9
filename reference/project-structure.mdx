The CV Builder is a client-side single-page application bootstrapped with Vite. All application code lives under `src/app/`, which is organized into clearly separated concerns: components, pages, schemas, services, store, and types. The root `App.tsx` orchestrates which view is rendered based on two pieces of local state — the active `View` and the selected `Template` — combined with the `selectedProfileId` and `showForm` values from the Zustand store.

## App.tsx — root component

`App.tsx` is the single entry point for view routing. It does not use React Router for page transitions; instead, it manages a `View` union type (`'welcome' | 'editor' | 'about'`) in local state and renders the appropriate component tree conditionally.

```typescript src/app/App.tsx
type View = 'welcome' | 'editor' | 'about';
type Template = 'modern' | 'classic';
```

The rendering logic follows this decision tree:

- No `selectedProfile` and `showForm` is false → render `<ProfileSelector />`
- `showForm` is true → render `<ProfileForm />`
- `selectedProfile` exists and `view === 'welcome'` → render `<Dashboard />`
- `selectedProfile` exists and `view === 'editor'` → render `<EditorPage />`
- `selectedProfile` exists and `view === 'about'` → render `<AboutPage />`

`TopBar` is only rendered when a profile is selected and the form is not visible. When `selectedProfileId` is cleared (e.g., via `goBack`), a `useEffect` resets the view to `'welcome'` automatically.

Both templates live in `src/app/components/preview/` and receive a `Profile` as their primary prop. They render the CV structure as HTML, which `exportService.exportToPDF` captures via `html2pdf.js`.

## Key files

### src/app/store/profile-store.ts

The Zustand store is the single source of truth for profile data at runtime. It holds the `profiles` array, the `selectedProfileId`, and the `showForm` flag that controls form visibility.

```typescript src/app/store/profile-store.ts
interface ProfileStore {
  profiles: Profile[];
  selectedProfileId: string | null;
  showForm: boolean;
  addProfile: (profile: Profile) => void;
  selectProfile: (profileId: string) => void;
  openForm: () => void;
  closeForm: () => void;
  goBack: () => void;
}

export const useProfileStore = create<ProfileStore>((set) => ({
  profiles: [],
  selectedProfileId: null,
  showForm: false,
  addProfile: (profile) =>
    set((state) => ({
      profiles: [...state.profiles, profile],
      showForm: false,
    })),
  selectProfile: (profileId) => set({ selectedProfileId: profileId }),
  openForm: () => set({ showForm: true, selectedProfileId: null }),
  closeForm: () => set({ showForm: false }),
  goBack: () => set({ selectedProfileId: null }),
}));
```

Store actions at a glance:

| Action | Behavior |
|--------|----------|
| `addProfile(profile)` | Appends a new `Profile` to `profiles` and sets `showForm` to `false` |
| `selectProfile(profileId)` | Sets `selectedProfileId` to activate a profile |
| `openForm()` | Sets `showForm: true` and clears `selectedProfileId` |
| `closeForm()` | Sets `showForm: false` without changing the selected profile |
| `goBack()` | Clears `selectedProfileId`, which triggers the `useEffect` in `App.tsx` to reset the view to `'welcome'` and return to `ProfileSelector` |

<Note>
  Profile data is held in memory only. There is no `localStorage` persistence or backend sync — refreshing the page resets the store to its initial empty state.
</Note>

### src/app/services/exportService.ts

`exportService.ts` exposes two export functions. PDF export uses `html2pdf.js` to render a DOM element to a downloadable A4 PDF. JSON export serializes the `Profile` object to a pretty-printed JSON file using the Blob and object URL APIs.

```typescript src/app/services/exportService.ts
export async function exportToPDF(elementId: string, filename = 'CV.pdf') {
  const element = document.getElementById(elementId);
  if (!element) return;

  await html2pdf()
    .set({
      margin: 10,
      filename,
      html2canvas: { scale: 2, useCORS: true },
      jsPDF: { orientation: 'portrait', unit: 'mm', format: 'a4' },
    })
    .from(element)
    .save();
}

export function exportToJSON(profile: Profile, filename = 'perfil-cv.json') {
  const blob = new Blob([JSON.stringify(profile, null, 2)], { type: 'application/json' });
  const url = URL.createObjectURL(blob);
  const link = document.createElement('a');
  link.href = url;
  link.download = filename;
  link.click();
  URL.revokeObjectURL(url);
}
```

`exportToPDF` renders at `scale: 2` for high-DPI output and targets A4 portrait in millimeters. `exportToJSON` immediately revokes the object URL after triggering the download to avoid memory leaks.
